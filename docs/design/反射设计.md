# C^ 静态反射设计文档

## 1. 设计原则

C^ 的反射机制是**纯静态（Compile-time）**的，遵循以下原则：

1. **零标准库依赖** — `@` 操作符与 `comptime for` 是纯编译器内置特性，不引用任何标准库模块
2. **元数据类型全用内置类型** — 元对象的所有属性类型均为编译器内置类型，不涉及标准库
3. **类型是一等公民** — `@T.fields[0].type` 返回的是真正的**编译期类型值**，不是字符串
4. **编译期/运行时严格分离** — 静态反射仅在编译期；运行时 RTTI 是独立的 opt-in 机制
5. **自定义属性系统** — 用户可以定义自己的属性，用于元编程

## 2. 编译器内置元类型

反射系统使用的所有类型都是**编译器内置**的，不依赖标准库：

### 2.1 `typeinfo` — 类型元信息

```c^
// 编译器内置类型，用户不能直接实例化
builtin typeinfo {
    literalview name;          // 类型名
    long size;                 // sizeof
    long align;                // alignof
    bool is_struct;
    bool is_class;
    bool is_enum;
    bool is_primitive;
    bool is_integer;
    bool is_float;
    bool is_pointer;
    bool is_reference;
    bool is_nullable;
}
```

### 2.2 `fieldinfo` — 字段元信息

```c^
builtin fieldinfo {
    literalview name;          // 字段名
    typeinfo type;             // 字段类型（编译期类型值）
    long offset;               // 字节偏移
    bool is_public;
    bool is_mutable;
    bool is_static;
}
```

### 2.3 `methodinfo` — 方法元信息

```c^
builtin methodinfo {
    literalview name;
    typeinfo return_type;
    int param_count;
    bool is_public;
    bool is_static;
    bool is_virtual;
    bool is_abstract;
}
```

### 2.4 `attributeinfo` — 属性元信息

```c^
builtin attributeinfo {
    literalview name;          // 属性类型名
    typeinfo type;             // 属性类型
}
```

### 2.5 `literalview` — 编译期字符串视图

```c^
// 编译器内置类型，表示编译期字符串
builtin literalview {
    // 零拷贝视图，指向源代码中的字符串字面量
    // 可在编译期比较、拼接
    func length() -> long;
    func equals(literalview other) -> bool;
    func starts_with(literalview prefix) -> bool;
    func ends_with(literalview suffix) -> bool;
}
```

> **重要**：`typeinfo`、`fieldinfo`、`methodinfo`、`attributeinfo`、`literalview` 都是编译器内置类型，不是标准库类型。它们只能在 `comptime` 上下文中使用。

## 3. 核心语法

### 3.1 `@` 操作符 — 获取类型的编译期元对象

```c^
@User              // 获取 User 类型的元对象，返回 typeinfo
@typeof(expr)      // 先取表达式的静态类型，再取该类型的元对象
@int               // 内置类型同样支持
```

`@T` 返回一个 `typeinfo` 值，只能在 `comptime` 上下文中使用。

### 3.2 元对象的属性访问

```c^
comptime {
    var ti = @User;
    
    // 类型基本信息
    ti.name           // literalview  — 类型名
    ti.size           // long         — 字节数
    ti.align          // long         — 对齐字节数
    ti.is_struct      // bool
    ti.is_class       // bool
    ti.is_enum        // bool
    ti.is_primitive   // bool
    
    // 成员访问
    ti.field_count    // int          — 字段数量
    ti.method_count   // int          — 方法数量
}
```

### 3.3 `typeof` — 获取表达式的静态类型

```c^
int a = 10;
typeof(a) b = 20;             // 等价于 int b = 20;

// 在 comptime 中使用
comptime if (@typeof(a).is_integer) {
    // a 是整数类型
}
```

## 4. `comptime for` — 编译期遍历

`comptime for` 遍历类型的成员，编译器将循环**完全展开**：

```c^
func print_fields<T>(T obj) {
    comptime for (var i = 0; i < @T.field_count; i++) {
        comptime var field = @T.field(i);
        print(field.name);
        print(": ");
        print(obj.[field]);
        print("\n");
    }
}
```

> `comptime for` 是编译期展开，**不是运行时循环**。

### 4.1 字段遍历

```c^
comptime for (var i = 0; i < @T.field_count; i++) {
    comptime var field = @T.field(i);
    // field 是 fieldinfo 类型
    // field.name  — 字段名
    // field.type  — 字段类型
    // field.offset — 字节偏移
}
```

### 4.2 方法遍历

```c^
comptime for (var i = 0; i < @T.method_count; i++) {
    comptime var method = @T.method(i);
    // method 是 methodinfo 类型
    // method.name — 方法名
    // method.is_static — 是否静态方法
}
```

---

## 5. 字段访问语法 `obj.[field]`

通过字段元对象访问对象的成员：

```c^
comptime var field = @Point.field(0);
obj.[field]           // 读取字段值
obj.[field] = value   // 写入字段
```

`obj.[field]` 与 `obj.field_name` 完全等价，区别仅在于字段名来自编译期元对象。

---

## 6. `comptime if` — 编译期条件分支

```c^
comptime for (var i = 0; i < @T.field_count; i++) {
    comptime var field = @T.field(i);
    
    comptime if (field.type.is_integer) {
        print_int(obj.[field]);
    } else comptime if (field.type.is_float) {
        print_float(obj.[field]);
    } else comptime if (field.type.equals(@literalview)) {
        print_str(obj.[field]);
    } else {
        // 递归处理嵌套结构体
        print_fields(obj.[field]);
    }
}
```

## 7. 自定义属性系统

属性（Attribute）是附加在声明上的编译期元数据，用于元编程。

### 7.1 属性定义

属性使用 `attribute` 关键字定义，所有字段必须是编译期常量类型：

```c^
// 无参数属性
attribute Serializable {}

// 带参数属性
attribute Column {
    literalview name;
    bool nullable = false;
}

// 带多个参数的属性
attribute JsonProperty {
    literalview name = "";
    bool ignore = false;
    int order = -1;
}

// 属性可以继承
attribute Validate : Attribute {
    literalview pattern;
    long min = 0;
    long max = Long.MAX_VALUE;
}
```

### 7.2 属性使用

属性可以附加在类型、字段、方法、参数等声明上：

```c^
[Serializable]
class User {
    [Column("id", nullable: false)]
    public int id;

    [Column("user_name")]
    [JsonProperty("userName")]
    public String name;

    [JsonProperty(ignore: true)]
    public int cached_hash;
    
    [Validate(pattern: "^[a-z]+$", min: 3, max: 20)]
    public String username;
}
```

### 7.3 属性读取

在 `comptime` 中读取属性：

```c^
func to_json<T>(T obj) -> String {
    String result = "{";
    bool first = true;
    
    comptime for (var i = 0; i < @T.field_count; i++) {
        comptime var field = @T.field(i);
        
        // 检查是否有 JsonProperty 属性
        comptime if (field.has_attribute(JsonProperty)) {
            comptime var attr = field.get_attribute(JsonProperty);
            
            // 检查是否忽略
            comptime if (attr.ignore) {
                continue;
            }
            
            if (!first) result += ",";
            first = false;
            
            result += "\"";
            // 使用属性中的名称，或字段名
            comptime if (attr.name.length() > 0) {
                result += attr.name;
            } else {
                result += field.name;
            }
            result += "\":";
            
            // 序列化值
            result += serialize_value(obj.[field]);
        }
    }
    
    result += "}";
    return result;
}
```

### 7.4 属性 API

```c^
// 检查字段是否有某属性
field.has_attribute(AttributeType) -> bool

// 获取属性实例
field.get_attribute(AttributeType) -> AttributeType

// 检查类型是否有某属性
@T.has_attribute(AttributeType) -> bool

// 获取类型的属性实例
@T.get_attribute(AttributeType) -> AttributeType

// 获取字段的所有属性
field.attribute_count -> int
field.attribute(i) -> attributeinfo
```

## 8. 编译器内置属性

编译器提供一些内置属性：

### 8.1 `[Reflectable]` — 启用运行时 RTTI

```c^
[Reflectable]
class Animal {
    public String name;
}

func main() {
    Animal a = Animal("Dog");
    // 运行时可以获取类型信息
    print(a.__rtti.name);  // "Animal"
}
```

### 8.2 `[Deprecated]` — 标记废弃

```c^
[Deprecated("Use newFunc instead")]
func oldFunc() { ... }
```

### 8.3 `[Inline]` / `[NoInline]` — 内联控制

```c^
[Inline]
func fast_add(int a, int b) -> int => a + b;

[NoInline]
func complex_operation() { ... }
```

### 8.4 `[Export]` — 导出符号

```c^
[Export]
public func api_function() -> int { ... }
```

## 9. 完整示例

### 9.1 ORM 映射（零标准库依赖）

```c^
attribute Table {
    literalview name;
}

attribute Column {
    literalview name = "";
    bool primary_key = false;
    bool nullable = true;
    literalview default_value = "";
}

[Table("users")]
class User {
    [Column(primary_key: true)]
    public long id;

    [Column("username", nullable: false)]
    public String name;

    [Column("email", nullable: false)]
    public String email;

    [Column(default_value: "NOW()")]
    public DateTime created_at;
}

func create_table<T>() -> String {
    comptime var table_attr = @T.get_attribute(Table);
    String sql = "CREATE TABLE " + table_attr.name + " (";
    
    comptime for (var i = 0; i < @T.field_count; i++) {
        comptime var field = @T.field(i);
        comptime var col = field.get_attribute(Column);
        
        if (i > 0) sql += ", ";
        
        // 列名
        comptime if (col.name.length() > 0) {
            sql += col.name;
        } else {
            sql += field.name;
        }
        
        // 类型映射
        comptime if (field.type.equals(@long)) {
            sql += " BIGINT";
        } else comptime if (field.type.equals(@int)) {
            sql += " INTEGER";
        } else comptime if (field.type.equals(@String)) {
            sql += " VARCHAR(255)";
        }
        
        // 约束
        comptime if (col.primary_key) {
            sql += " PRIMARY KEY";
        }
        comptime if (!col.nullable) {
            sql += " NOT NULL";
        }
        comptime if (col.default_value.length() > 0) {
            sql += " DEFAULT " + col.default_value;
        }
    }
    
    sql += ");";
    return sql;
}

// 使用
comptime {
    print(create_table<User>());
    // 输出: CREATE TABLE users (id BIGINT PRIMARY KEY, username VARCHAR(255) NOT NULL, email VARCHAR(255) NOT NULL, created_at DATETIME DEFAULT NOW());
}
```

### 9.2 序列化框架

```c^
attribute Serialize {
    literalview name = "";
    int order = -1;
    bool skip = false;
}

func serialize<T>(T obj) -> String {
    // 按 order 排序字段
    String result = "{";
    
    comptime for (var i = 0; i < @T.field_count; i++) {
        comptime var field = @T.field(i);
        comptime var attr = field.get_attribute(Serialize);
        
        comptime if (attr.skip) continue;
        
        if (i > 0) result += ",";
        
        result += "\"";
        comptime if (attr.name.length() > 0) {
            result += attr.name;
        } else {
            result += field.name;
        }
        result += "\":";
        
        result += serialize_value(obj.[field]);
    }
    
    result += "}";
    return result;
}
```

## 10. 与标准库的关系

### 10.1 零依赖原则

静态反射系统**完全不依赖标准库**：

| 特性 | 实现方式 |
|---|---|
| `typeinfo` | 编译器内置类型 |
| `fieldinfo` | 编译器内置类型 |
| `methodinfo` | 编译器内置类型 |
| `attributeinfo` | 编译器内置类型 |
| `literalview` | 编译器内置类型 |
| `@` 操作符 | 编译器内置语法 |
| `comptime` | 编译器内置关键字 |
| `attribute` | 编译器内置关键字 |

### 10.2 标准库可选使用

标准库可以基于反射系统提供高级功能，但反射本身不依赖标准库：

```c^
// 标准库可以这样实现（用户也可以自己实现）
module std.json {
    func to_json<T>(T obj) -> String {
        // 使用反射实现
    }
}

// 用户也可以自己实现，不需要导入标准库
func my_to_json<T>(T obj) -> String {
    // 同样可以使用反射
}
```

## 11. 运行时反射（严格 opt-in）

静态反射与运行时反射**完全独立**。

### 11.1 启用

```c^
[Reflectable]
class Animal {
    public String name;
    public int age;
}
```

### 11.2 使用

```c^
Animal a = Animal("Dog", 5);

// 运行时类型信息
print(a.__rtti.name);           // "Animal"
print(a.__rtti.field_count);    // 2

// 运行时字段访问
comptime for (var i = 0; i < @Animal.field_count; i++) {
    comptime var field = @Animal.field(i);
    print(field.name);
    print(": ");
    print(a.__get_field(field.name));  // 运行时字段访问
}
```

### 11.3 原则

- 运行时反射需要显式启用 `[Reflectable]`
- 默认关闭，不产生任何二进制开销
- 运行时反射的 `__rtti` 是编译器注入的隐藏字段

## 12. 总结

| 特性 | 说明 |
|---|---|
| `@T` | 获取类型 T 的编译期元对象，返回 `typeinfo` |
| `@typeof(expr)` | 获取表达式静态类型的元对象 |
| `typeinfo` | 编译器内置类型，包含类型基本信息 |
| `fieldinfo` | 编译器内置类型，包含字段信息 |
| `methodinfo` | 编译器内置类型，包含方法信息 |
| `literalview` | 编译器内置字符串视图类型 |
| `obj.[field]` | 通过字段元对象访问成员 |
| `comptime for` | 编译期展开循环 |
| `comptime if` | 编译期条件分支 |
| `attribute` | 定义自定义属性 |
| `[Attr]` | 使用属性 |
| `field.has_attribute(Attr)` | 检查是否有属性 |
| `field.get_attribute(Attr)` | 获取属性实例 |
| `[Reflectable]` | opt-in 运行时 RTTI |
| **标准库依赖** | **无**，全部内置于编译器 |
