# C^ 静态反射设计文档

## 1. 设计原则

C^ 的反射机制是**纯静态（Compile-time）**的，遵循以下原则：

1. **零标准库依赖** — `@` 操作符与 `comptime for` 是纯编译器内置特性，不引用任何 `std.*` 模块
2. **元数据类型全用内置类型** — 元对象的所有属性类型均为 `literalview`、`bool`、`int`、`long`，不涉及标准库类型
3. **类型是一等公民** — `@T.fields[0].type` 返回的是真正的**编译期类型值**，不是字符串
4. **编译期/运行时严格分离** — 静态反射仅在编译期；运行时 RTTI 是独立的 opt-in 机制（`[Reflectable]`）

---

## 2. 核心语法

### 2.1 `@` 操作符 — 获取类型的编译期元对象

```c^
@User              // 获取 User 类型的元对象
@typeof(expr)      // 先取表达式的静态类型，再取该类型的元对象
@int               // 内置类型同样支持
```

`@T` 返回一个**编译器生成的匿名元对象**，其类型不对用户可见，只能在 `comptime` 上下文中使用。

> ⚠️ `@` 永远作用于**类型**，不能直接作用于值：
> - `@User` — 合法
> - `@typeof(a)` — 合法，`typeof(a)` 先求类型
> - `@a` — **编译错误**，`a` 是值

### 2.2 元对象的内置属性

所有属性均为编译期常量，类型全部是内置类型：

```c^
// 类型基本信息
@T.name            // literalview  — 类型名，如 "User"
@T.is_struct       // bool
@T.is_class        // bool
@T.is_enum         // bool
@T.is_primitive    // bool         — int/bool/byte/char/long/float/double
@T.size            // long         — sizeof(T)，字节数
@T.align           // long         — alignof(T)，对齐字节数

// 成员信息
@T.field_count     // int          — 字段数量
@T.fields          // 编译期异构元组，只能用 comptime for 遍历
@T.method_count    // int          — 方法数量
@T.methods         // 编译期异构元组，只能用 comptime for 遍历
```

**字段元对象的属性：**

```c^
field.name         // literalview  — 字段名
field.type         // 类型值        — 编译期类型，可与 typeof() 比较
field.offset       // long         — 相对结构体起始的字节偏移
field.is_public    // bool
field.is_mutable   // bool         — var 字段为 true，let 字段为 false
field.has_attr(A)  // bool         — 是否有 Attribute A
field.attr(A)      // A 的编译期实例，仅当 has_attr 为 true 时有效
```

**方法元对象的属性：**

```c^
method.name        // literalview
method.is_public   // bool
method.is_static   // bool
method.param_count // int
```

### 2.3 `typeof` — 获取表达式的静态类型

`typeof` 是关键字，返回编译期类型值，可用于变量声明、泛型参数、类型比较：

```c^
int a = 10;
typeof(a) b = 20;             // 等价于 int b = 20;
List<typeof(a)> list;         // 等价于 List<int>

// 与字段类型比较
comptime if (field.type == typeof(int)) { ... }
```

## 3. `comptime for` — 编译期异构展开

`comptime for` 遍历 `@T.fields` 或 `@T.methods`，编译器将循环**完全展开**为独立语句序列，不产生任何运行时循环：

```c^
func print_fields<T>(T! obj) {
    comptime for (var field : @T.fields) {
        // 每次迭代 field 是不同的编译期常量
        // 循环体必须对所有字段类型都合法
        print(field.name);
        print(": ");
        print(obj.[field]);
        print("\n");
    }
}
```

> `comptime for` 是编译期展开，**不是运行时循环**。`field` 的类型在每次展开时可以不同（异构）。

---

## 4. 字段访问语法 `obj.[field]`

通过字段元对象访问对象的成员：

```c^
obj.[field]           // 读取字段值，类型为 field.type（编译期已知）
obj.[field] = value   // 写入字段（仅 field.is_mutable == true 时合法）
```

`obj.[field]` 与 `obj.name`（硬编码字段名）完全等价，区别仅在于字段名来自编译期元对象。

---

## 5. `comptime if` — 编译期条件分支

配合字段类型做分支，实现多态编译期处理：

```c^
comptime for (var field : @T.fields) {
    comptime if (field.type == typeof(int) || field.type == typeof(long)) {
        print_int(obj.[field]);
    } else comptime if (field.type == typeof(bool)) {
        print_bool(obj.[field]);
    } else comptime if (field.type == typeof(literalview)) {
        print_str(obj.[field]);
    } else {
        // 递归处理嵌套结构体
        print_fields(obj.[field]);
    }
}
```

## 6. Attribute — 自定义编译期元数据

允许给字段或类型附加用户定义的编译期元数据，不依赖运行时。

### 6.1 定义 Attribute

```c^
// Attribute 是 comptime struct，字段全部是编译期常量类型
comptime struct Column {
    literalview name;
    bool nullable = false;
}

comptime struct Ignore {}
```

### 6.2 使用 Attribute

```c^
class User {
    [Column("id")]
    public int id;

    [Column("user_name")]
    public String name;

    [Ignore]
    public int cached_hash;
}
```

### 6.3 读取 Attribute

```c^
comptime for (var field : @User.fields) {
    comptime if (field.has_attr(Ignore)) {
        // 跳过此字段
    } else comptime if (field.has_attr(Column)) {
        comptime var col = field.attr(Column);
        // col.name、col.nullable 都是编译期常量
        generate_column(col.name, col.nullable);
    }
}
```

## 7. 完整示例

### 7.1 JSON 序列化（零标准库依赖）

```c^
// 仅依赖内置类型，String 是标准库但可替换为任意输出方式
func to_json<T>(T! obj) -> String {
    String buf = "{";
    comptime for (var field : @T.fields) {
        comptime if (field.name != @T.fields[0].name) {
            buf += ",";
        }
        buf += "\"";
        buf += field.name;
        buf += "\":";
        comptime if (field.type == typeof(int) || field.type == typeof(long)) {
            buf += to_string(obj.[field]);
        } else comptime if (field.type == typeof(bool)) {
            buf += obj.[field] ? "true" : "false";
        } else comptime if (field.type == typeof(literalview)) {
            buf += "\"";
            buf += obj.[field];
            buf += "\"";
        } else {
            buf += to_json(obj.[field]);   // 递归
        }
    }
    buf += "}";
    return buf;
}

// 使用
struct Point { int x; int y; }
var p = Point(1, 2);
print(to_json(p));  // {"x":1,"y":2}
```

### 7.2 字段名枚举

```c^
func field_names<T>() -> void {
    comptime for (var field : @T.fields) {
        print(field.name);   // literalview，零开销
        print(" ");
    }
}
```

### 7.3 编译期断言结构体布局

```c^
comptime {
    static_assert(@Point.field_count == 2, "Point must have 2 fields");
    static_assert(@Point.fields[0].type == typeof(int), "first field must be int");
    static_assert(@Point.size == 8, "Point must be 8 bytes");
}
```

## 8. 运行时反射（严格 opt-in）

静态反射与运行时反射**完全独立**，不共用任何机制。

### 8.1 启用

```c^
[Reflectable]   // 显式标注，编译器注入 rtti 字段
class Animal { ... }

[Reflectable]
class Dog : Animal { ... }
```

### 8.2 使用

```c^
Animal^ a = new Dog;
a.rtti.name         // literalview，运行时可用，值为 "Dog"（动态类型名）
```

对于未标注 `[Reflectable]` 的类型，`rtti` 字段不存在，访问即编译错误。

### 8.3 原则

- 运行时反射不依赖静态反射的 `@` 操作符
- 不生成全局类型表，只为 `[Reflectable]` 类型注入最小 RTTI 字段
- 默认关闭，不产生任何二进制开销

---

## 9. 总结

| 特性 | 说明 |
|---|---|
| `@T` | 获取类型 T 的编译期元对象 |
| `@typeof(expr)` | 获取表达式静态类型的元对象 |
| `@T.fields` | 字段异构元组，只能 `comptime for` 遍历 |
| `@T.methods` | 方法异构元组，只能 `comptime for` 遍历 |
| `obj.[field]` | 通过字段元对象访问成员 |
| `field.type == typeof(T)` | 编译期类型比较 |
| `comptime for` | 编译期展开循环，生成独立语句 |
| `comptime if` | 编译期条件分支，零运行时开销 |
| `[Attribute]` | 自定义编译期元数据，`comptime struct` 定义 |
| `[Reflectable]` | opt-in 运行时 RTTI，默认关闭 |
| 标准库依赖 | **无**，全部内置于编译器 |
