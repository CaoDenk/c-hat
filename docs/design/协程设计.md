# C^ 协程设计文档 (Coroutines)

## 1. 设计哲学

C^ 的协程设计灵感来源于 C++20 的无栈协程（Stackless Coroutines），但旨在提供更清晰、更优雅的**契约式（Duck Typing）**编程模型。

*   **无栈 (Stackless)**：协程状态保存在堆（或自定义分配器）上的状态机中，而非占用线程栈。
*   **零开销抽象**：如果编译器能推导生命周期，甚至可以优化为状态机内联。
*   **契约驱动**：协程的行为（是 Generator 还是 Task？是 Lazy 还是 Eager？）完全由**返回类型**的契约决定，而非硬编码在编译器中。
*   **零标准库依赖**：协程核心机制是编译器内置特性，不依赖标准库。

---

## 2. 编译器内置类型

协程系统使用的核心类型都是**编译器内置**的，不依赖标准库：

### 2.1 `CoroutineHandle` — 协程句柄

```c^
// 编译器内置类型，表示协程的句柄
builtin struct CoroutineHandle {
    // 从 Promise 创建句柄
    static func from_promise<T>(T promise) -> CoroutineHandle;
    
    // 恢复协程执行
    func resume();
    
    // 销毁协程
    func destroy();
    
    // 检查协程是否完成
    func done() -> bool;
    
    // 获取 Promise 对象
    func promise<T>() -> T;
}
```

### 2.2 `SuspendAlways` — 始终挂起

```c^
// 编译器内置 Awaitable，始终挂起
builtin struct SuspendAlways {
    func is_ready() -> bool => false;
    func on_suspend(CoroutineHandle) {}
    func on_resume() {}
}
```

### 2.3 `SuspendNever` — 从不挂起

```c^
// 编译器内置 Awaitable，从不挂起
builtin struct SuspendNever {
    func is_ready() -> bool => true;
    func on_suspend(CoroutineHandle) {}
    func on_resume() {}
}
```

> **重要**：`CoroutineHandle`、`SuspendAlways`、`SuspendNever` 是编译器内置类型，不是标准库类型。

---

## 3. 关键字

两个核心关键字驱动协程状态机：

*   **`await`**：一元操作符。挂起当前协程，等待操作对象（Awaitable）完成，然后恢复，表达式结果为 Awaitable 的 `on_resume()` 返回值。
*   **`yield`**：语句。挂起当前协程，向调用者产出一个值（用于生成器）。

> **注意**：C^ 不需要显式的 `async` 关键字标记函数。如果一个函数体内包含 `await` 或 `yield`，且其返回类型满足协程契约，它**自动成为协程**。

## 4. 协程契约 (Coroutine Contract)

C^ 编译器并不认识具体的 `Task` 或 `Generator` 类，它只认识**模式**。

### 4.1 Promise 类型查找规则

**唯一规则**：编译器在返回类型内查找名为 `Promise` 的内嵌类型。

```c^
struct Task {
    struct Promise { ... }   // 编译器找 Task.Promise
}

struct Generator<T> {
    struct Promise { ... }   // 编译器找 Generator<T>.Promise
}
```

不支持外部挂载 Promise（类似 C++ `coroutine_traits`）——这和 `extension` 不能实现接口是同一原则，避免隐式的类型关系污染。

### 4.2 Promise 必须实现的契约

```c^
struct Task {
    struct Promise {
        // 1. 创建协程对象：协程启动时调用，返回外层的 Task 对象
        func get_return_object() -> Task;

        // 2. 初始挂起点：协程创建后是立即执行还是先挂起？
        //    返回 SuspendAlways → 冷启动（Lazy，像 Rust/Python）
        //    返回 SuspendNever  → 热启动（Eager，像 C#）
        func initial_suspend() -> SuspendAlways;

        // 3. 最终挂起点：协程结束后挂起，保持 handle 有效
        func final_suspend() -> SuspendAlways;

        // 4a. 处理返回值（对应 `return value;`）
        func return_value(int value);
        // 4b. 或者无返回值版本（对应 `return;` 或函数自然结束）
        // func return_void();

        // 5. 处理未捕获异常
        func unhandled_exception();

        // 6. （可选）处理 yield（对应 `yield value;`）
        // func yield_value(T value) -> SuspendAlways;
    }
}
```

### 4.3 Awaitable 契约

任何可以被 `await` 的对象必须满足以下契约。`await expr` 的结果类型由 `on_resume()` 的返回类型决定：

```c^
// Result 是关联类型，由具体 Awaitable 决定
struct MyAwaitable {
    // 1. 是否已就绪？返回 true 则不挂起，直接取结果
    func is_ready() -> bool;

    // 2. 挂起逻辑：将协程句柄注册到事件循环/线程池
    func on_suspend(CoroutineHandle handle);

    // 3. 恢复逻辑：await 表达式的求值结果
    //    返回类型由 Awaitable 自身决定（可以是任意类型）
    func on_resume() -> int;   // 此处 int 只是示例，实际随 Awaitable 而定
}
```

对于泛型 Awaitable：

```c^
struct TaskAwaitable<T> {
    func is_ready() -> bool;
    func on_suspend(CoroutineHandle handle);
    func on_resume() -> T;    // await 结果类型为 T
}
```

## 5. 示例实现

### 5.1 简单的生成器 (Generator) — 零标准库依赖

```c^
// 完全不依赖标准库的生成器实现
struct Generator {
    struct Promise {
        int current_value;

        func get_return_object() -> Generator {
            return Generator(CoroutineHandle.from_promise(self));
        }

        func initial_suspend() -> SuspendAlways => {};
        func final_suspend()   -> SuspendAlways => {};
        func return_void() {}
        func unhandled_exception() { /* 用户自定义错误处理 */ }

        func yield_value(int v) -> SuspendAlways {
            self.current_value = v;
            return {};
        }
    }

    CoroutineHandle handle;

    // 下一个值
    func next() -> int? {
        if (handle.done()) return null;
        handle.resume();
        if (handle.done()) return null;
        return handle.promise<Promise>().current_value;
    }
    
    // 析构时销毁协程
    func ~Generator() {
        if (handle) handle.destroy();
    }
}

// 使用
func range(int start, int end) -> Generator {
    for (int i = start; i < end; i++) {
        yield i;
    }
}

func main() {
    var gen = range(0, 5);
    while (var val = gen.next()) {
        print(val);
    }
}
```

### 5.2 泛型生成器

```c^
struct Generator<T> {
    struct Promise {
        T current_value;

        func get_return_object() -> Generator<T> {
            return Generator<T>(CoroutineHandle.from_promise(self));
        }

        func initial_suspend() -> SuspendAlways => {};
        func final_suspend()   -> SuspendAlways => {};
        func return_void() {}
        func unhandled_exception() {}

        func yield_value(T v) -> SuspendAlways {
            self.current_value = v;
            return {};
        }
    }

    CoroutineHandle handle;

    func next() -> T? {
        if (handle.done()) return null;
        handle.resume();
        if (handle.done()) return null;
        return handle.promise<Promise>().current_value;
    }
    
    func ~Generator() {
        if (handle) handle.destroy();
    }
}

// 使用
func fibonacci(int n) -> Generator<long> {
    long a = 0, b = 1;
    for (int i = 0; i < n; i++) {
        yield a;
        long temp = a + b;
        a = b;
        b = temp;
    }
}
```

### 5.3 异步任务 (Async Task) — 零标准库依赖

```c^
// 简单的异步任务，不依赖标准库
struct Task<T> {
    struct Promise {
        T result;
        bool has_result = false;
        CoroutineHandle waiter;

        func get_return_object() -> Task<T> {
            return Task<T>(CoroutineHandle.from_promise(self));
        }

        func initial_suspend() -> SuspendNever => {};  // 热启动
        func final_suspend() -> SuspendAlways => {};
        
        func return_value(T v) {
            self.result = v;
            self.has_result = true;
            if (waiter) waiter.resume();
        }
        
        func unhandled_exception() { /* 用户自定义 */ }
    }

    CoroutineHandle handle;

    // 同步等待结果
    func sync_wait() -> T {
        if (!handle.done()) {
            handle.promise<Promise>().waiter = CoroutineHandle.from_promise(self);
            SuspendAlways{}.on_suspend(handle);
        }
        return handle.promise<Promise>().result;
    }
    
    // Awaitable 支持
    struct Awaitable {
        Task<T> task;
        
        func is_ready() -> bool => task.handle.done();
        
        func on_suspend(CoroutineHandle waiter) {
            task.handle.promise<Promise>().waiter = waiter;
        }
        
        func on_resume() -> T => task.handle.promise<Promise>().result;
    }
    
    func await_get() -> Awaitable => Awaitable{task: self};
}

// 使用
func fetch_data(String url) -> Task<String> {
    // 模拟异步操作
    return "Response from " + url;
}

func main() {
    var task = fetch_data("http://example.com");
    print(task.sync_wait());
}
```

## 6. 协程状态机

编译器将协程函数转换为状态机：

### 6.1 状态机结构

```c^
// 源代码
func example() -> Generator {
    yield 1;
    yield 2;
    yield 3;
}

// 编译器生成的状态机（概念）
struct example_StateMachine {
    int state = 0;  // 0=初始, 1=第一个yield后, 2=第二个yield后, 3=完成
    Generator.Promise* promise;
    
    func resume() {
        switch (state) {
            case 0:
                state = 1;
                promise.yield_value(1);
                return;
            case 1:
                state = 2;
                promise.yield_value(2);
                return;
            case 2:
                state = 3;
                promise.yield_value(3);
                return;
            case 3:
                promise.return_void();
                return;
        }
    }
}
```

### 6.2 状态保存

协程挂起时，所有局部变量都保存在状态机中：

```c^
func complex_coroutine(int n) -> Generator<int> {
    int a = 0;      // 保存在状态机中
    int b = 1;      // 保存在状态机中
    for (int i = 0; i < n; i++) {
        yield a;    // 挂起点，a, b, i 都需要保存
        int temp = a + b;
        a = b;
        b = temp;
    }
}
```

## 7. 内存管理

### 7.1 协程帧分配

协程帧（保存状态机的内存）默认在堆上分配：

```c^
// 编译器生成的分配代码
func example() -> Generator {
    // 分配协程帧
    var frame = __coroutine_allocate(sizeof(example_frame));
    var promise = new (frame) Generator.Promise();
    // ...
}
```

### 7.2 自定义分配器

用户可以通过 Promise 提供自定义分配器：

```c^
struct Task {
    struct Promise {
        // 自定义分配
        func operator_new(long size) -> void* {
            return my_allocator.alloc(size);
        }
        
        // 自定义释放
        func operator_delete(void* ptr) {
            my_allocator.free(ptr);
        }
        
        // ... 其他 Promise 方法
    }
}
```

## 8. 与标准库的关系

### 8.1 零依赖原则

协程核心机制**完全不依赖标准库**：

| 特性 | 实现方式 |
|---|---|
| `CoroutineHandle` | 编译器内置类型 |
| `SuspendAlways` | 编译器内置类型 |
| `SuspendNever` | 编译器内置类型 |
| `await` | 编译器内置关键字 |
| `yield` | 编译器内置关键字 |
| Promise 契约 | 编译器内置规则 |
| 状态机生成 | 编译器内置功能 |

### 8.2 标准库可选使用

标准库可以基于协程机制提供高级功能，但协程本身不依赖标准库：

```c^
// 标准库可以这样实现（用户也可以自己实现）
module std.async {
    struct Task<T> {
        struct Promise { ... }
        // 高级功能：取消、超时、组合等
    }
    
    struct EventLoop {
        func run();
        func spawn<T>(Task<T> task);
    }
}

// 用户也可以自己实现，不需要导入标准库
struct MyTask<T> {
    struct Promise { ... }
    // 自定义实现
}
```

## 9. 设计总结

| 特性 | C++ | C^ |
|---|---|---|
| 关键字 | `co_await` / `co_yield` / `co_return` | `await` / `yield` / `return` |
| 函数标记 | 无需标记，有关键字自动识别 | 同左 |
| Promise 查找 | `coroutine_traits<Ret, Args...>` 或内嵌 `promise_type` | **只看内嵌 `Promise` 类型** |
| 外部挂载 Promise | 支持（`coroutine_traits` 特化） | **不支持**，避免隐式类型污染 |
| 内置 Awaitable | `std::suspend_always` / `std::suspend_never` | `SuspendAlways` / `SuspendNever`（编译器内置） |
| 协程句柄 | `std::coroutine_handle` | `CoroutineHandle`（编译器内置） |
| 标准库依赖 | 必须依赖 `<coroutine>` | **无**，全部内置于编译器 |

C^ 的协程设计优雅在：

1. **统一性**：`Generator` 和 `Task` 只是 `Promise` 实现不同，语法完全一致
2. **控制权下放**：`initial_suspend` 由库作者决定冷启动还是热启动
3. **规则极简**：Promise 查找只有一条规则，无历史包袱
4. **类型安全**：所有状态机生成都在编译期完成，`await` 结果类型由 `on_resume()` 静态决定
5. **零依赖**：核心机制全部内置于编译器，不依赖标准库
