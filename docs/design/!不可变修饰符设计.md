# C^ `!` 不可变修饰符设计文档

`!` 是 C^ 中统一的**不可变（Immutable / Readonly）修饰符**。它的核心规则只有一条：

> **`!` 是左绑定的**：它修饰**紧接在它左边**的类型部分，而不是整个类型表达式。

---

## 1. 绑定规则详解

### 1.1 基本绑定规则

```
类型写法     绑定解析         语义
──────────────────────────────────────────────────────────
int!         (int)!           不可变整数值（值本身不能修改）
int![]       (int!)[]         元素不可变的数组（可以重新绑定切片，但不能修改元素）
int[]!       (int[])!         整体不可变的切片（不能重新绑定切片，元素仍可修改）
int!^        (int!)^          指向不可变 int 的指针（通过该指针无法修改所指的值）
int^!        (int^)!          不可变指针（指针本身不能重新指向，但可以修改所指的值）
int!&        (int!)&          不可变引用（通过该引用无法修改绑定的值）
int![]^      (int![])^        指向"元素不可变数组"的指针
int![]!      (int!)[]!        整体不可变且元素不可变的切片
```

### 1.2 记忆方法

从左到右读类型，遇到 `!` 就在心里说"**左边这个是只读的**"：

- `int![]`：`int!` → int 是只读的，`[]` → 组成数组 → **元素只读的数组**
- `int[]!`：`int[]` → int 数组，`!` → 数组本身是只读的 → **切片整体不可重新绑定**

## 2. 各使用场景

### 2.1 值不可变

```cpp
int! x = 42;       // x 的值不能被修改（类似 C++ const int）
// x = 43;         // Error: 不可变变量

// 注意：let 关键字会自动附加 !，以下两种写法等价
let y = 42;        // 自动推导为 int!
int! y = 42;       // 显式写法
```

### 2.2 不可变引用

```cpp
// 形参：int!& 表示"不可变引用"，调用方的值不会被修改
func print_value(int!& x) {
    print(x);
    // x = 100;   // Error: 不可变引用，不能赋值
}

func main() {
    int a = 42;
    print_value(a);  // 推荐：隐式传递（无副作用，无需 & 标记）
}
```

### 2.3 指向不可变数据的指针 vs 不可变指针

```cpp
int value = 10;
int another = 20;

// 指向不可变 int 的指针：int!^
int!^ p1 = ^value;
// p1^ = 99;       // Error: 不能通过此指针修改值
p1 = ^another;     // OK: 指针本身可以重新指向

// 不可变指针：int^!
int^! p2 = ^value;
p2^ = 99;          // OK: 可以修改所指的值
// p2 = ^another;  // Error: 指针本身不能重新指向

// 双重不可变：int!^!
int!^! p3 = ^value;
// p3^ = 99;       // Error: 不能修改值
// p3 = ^another;  // Error: 不能重新指向
```

### 2.4 数组与切片的不可变性

```cpp
int[5] arr = {1, 2, 3, 4, 5};

// 元素不可变的切片：int![]
int![] read_only_data = arr[];
// read_only_data[0] = 99;   // Error: 元素不可修改
int![] another_slice = arr[1:3];  // OK: 切片本身可重新绑定

// 整体不可变的切片（不可重新绑定）：int[]!
int[]! fixed_slice = arr[];
fixed_slice[0] = 99;          // OK: 元素可以修改
// fixed_slice = arr[1:3];    // Error: 不能重新绑定切片

// 完全不可变：int![]!
int![]! frozen = arr[];
// frozen[0] = 99;            // Error
// frozen = arr[1:3];         // Error
```

### 2.5 不可变方法

成员函数有两种等价的写法来声明不可变（即不修改 `self`）：

```cpp
class Vec2 {
    float x;
    float y;

    // 写法一：函数后缀 !（推荐，简洁）
    func length()! -> float {
        return sqrt(x * x + y * y);
    }

    // 写法二：self! 参数（显式风格）
    func area(self!) -> float {
        return x * y;
    }
}
```

两种写法完全等价，可以混用。同时使用时编译器会发出警告（冗余）：

```cpp
func foo(self!)! { }  // Warning: 冗余的 !
```

## 3. 与 `const` / `let` 的关系

| 关键字 / 符号 | 作用域 | 语义 |
|-------------|--------|------|
| `const`     | 编译期 | 编译期常量，值必须在编译时确定 |
| `let`       | 运行时 | 运行时不可变变量，等价于类型加 `!` |
| `!`（类型修饰）| 类型系统 | 修饰引用、指针、切片等复合类型中某层的可变性 |

```cpp
const int MAX = 100;    // 编译期常量
let x = 42;             // 运行时不可变，等价于 int! x = 42
int!& r = &x;           // 不可变引用，指向不可变 int
```

## 4. 常见错误与陷阱

### 陷阱 1：误解 `int![]` 的含义

```cpp
// ❌ 错误理解：整个切片不可变
// ✅ 正确理解：元素不可变，但可以重新绑定切片变量
int![] s = arr[];
s = arr[1:3];    // OK：切片变量本身是可变的（s 是 var/普通变量）
s[0] = 1;        // Error：元素不可修改
```

如果想让切片变量本身也不可重新绑定，需要：

```cpp
int![]! s = arr[];   // 元素不可变 且 切片变量不可重新绑定
// 或者
let s = arr[];       // let 让 s 不可重新绑定，但不影响元素可变性（= int[]!）
```

### 陷阱 2：`int!^` vs `int^!`

最容易混淆的一对。记住：**`!` 修饰它左边的那层**。

```
int!^   →  (int!)^   →  指向只读 int 的指针（C++ 的 const int*）
int^!   →  (int^)!   →  只读的指向 int 的指针（C++ 的 int* const）
```

---

## 5. 与 C++ 的对照

| C^ 写法 | C++ 等价写法 | 语义 |
|--------|------------|------|
| `int!` | `const int` | 不可变值 |
| `int!&` | `const int&` | 不可变引用 |
| `int!^` | `const int*` | 指向常量的指针 |
| `int^!` | `int* const` | 常量指针 |
| `int!^!` | `const int* const` | 常量指针指向常量 |
| `int![]` | `const int[]`（切片语义） | 元素只读切片 |
