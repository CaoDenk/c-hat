# C^ 语言命名规范

## 概述

本文档定义 C^ 语言和编译器实现的命名规范，确保代码风格一致、易读、易维护。

---

## 1. 类型命名（Type Names）

### 1.1 内置类型

- **基本类型**：全小写
  - `void`
  - `bool`
  - `byte`、`sbyte`
  - `short`、`ushort`
  - `int`、`uint`
  - `long`、`ulong`
  - `float`、`double`
  - `fp16`、`bf16`
  - `char`

- **特殊内置类型**：PascalCase（大驼峰）
  - `LiteralView` - 字符串字面量类型
  - `StringView` - 字符串视图类型（`byte![]` 的别名）
  - `String` - 标准库字符串类型

### 1.2 用户定义类型

- **类/结构体/枚举**：PascalCase
  ```cpp
  class Person { ... }
  struct Point { ... }
  enum Color { ... }
  ```

- **类型别名**：PascalCase
  ```cpp
  using IntList = List<int>;
  ```

---

## 2. 函数命名（Function Names）

### 2.1 函数名

- **函数名**：camelCase（小驼峰）
  ```cpp
  func calculateSum(int a, int b) -> int { ... }
  func printMessage(StringView msg) { ... }
  ```

### 2.2 特殊函数

- **构造函数**：与类名相同（PascalCase）
  ```cpp
  class Person {
      Person(string name, int age) { ... }
  }
  ```

- **析构函数**：`~` + 类名
  ```cpp
  class Person {
      ~Person() { ... }
  }
  ```

- **操作符重载**：`operator` + 操作符
  ```cpp
  func operator +(int a, int b) -> int { ... }
  ```

---

## 3. 变量命名（Variable Names）

### 3.1 普通变量

- **变量名**：camelCase（小驼峰）
  ```cpp
  var userName = "Alice";
  let itemCount = 10;
  ```

### 3.2 成员变量

- **成员变量**：camelCase（小驼峰）
  ```cpp
  class Person {
      string name;
      int age;
  }
  ```

### 3.3 常量

- **编译期常量**：PascalCase 或 UPPER_SNAKE_CASE
  ```cpp
  const MaxCount = 100;
  const PI = 3.14159;
  ```

---

## 4. 文件命名（File Names）

### 4.1 源文件

- **头文件和实现文件**：PascalCase，与类/类型名对应
  ```
  ExternDecl.h
  ExternDecl.cpp
  FunctionDecl.h
  FunctionDecl.cpp
  ```

### 4.2 目录结构

- **目录名**：小写，使用下划线或连字符（根据现有约定）
  ```
  src/
    ast/
    lexer/
    parser/
    semantic/
    llvm/
  ```

---

## 5. 宏和枚举值命名（Macros & Enum Values）

### 5.1 枚举值

- **枚举值**：PascalCase
  ```cpp
  enum Color {
      Red,
      Green,
      Blue
  }
  ```

### 5.2 编译期常量

- **编译期常量**：PascalCase 或 UPPER_SNAKE_CASE

---

## 6. 命名空间命名（Namespace Names）

- **命名空间**：全小写
  ```cpp
  namespace c_hat { ... }
  namespace ast { ... }
  namespace parser { ... }
  ```

---

## 7. 模板和泛型命名（Template & Generic Names）

- **类型参数**：单个大写字母（T、U、V）或 PascalCase 描述性名称
  ```cpp
  class List<T> { ... }
  func map<T, U>(List<T> list, func(T) -> U f) -> List<U> { ... }
  ```

---

## 8. 缩略语（Acronyms）

- **缩略语**：视为单词，使用 PascalCase 或 camelCase
  - `HTML` → `Html`
  - `URL` → `Url`
  - `API` → `Api`
  - `JSON` → `Json`

  示例：
  ```cpp
  class HtmlParser { ... }
  func parseUrl(string url) -> Url { ... }
  ```

---

## 9. 布尔值命名（Boolean Names）

- **布尔变量/函数**：使用描述性前缀
  - `is`、`has`、`can`、`should`、`need` 等

  示例：
  ```cpp
  bool isEmpty;
  bool hasValue;
  func canExecute() -> bool { ... }
  ```

---

## 10. 总结

| 元素           | 规范         | 示例                           |
| :------------- | :----------- | :----------------------------- |
| 内置基本类型   | 全小写       | `int`、`byte`、`bool`          |
| 内置特殊类型   | PascalCase   | `LiteralView`、`StringView`    |
| 类/结构体/枚举 | PascalCase   | `Person`、`Point`、`Color`     |
| 函数名         | camelCase    | `calculateSum`、`printMessage` |
| 变量名         | camelCase    | `userName`、`itemCount`        |
| 成员变量       | camelCase    | `name`、`age`                  |
| 文件名         | PascalCase   | `ExternDecl.h`、`Parser.cpp`   |
| 命名空间       | 全小写       | `c_hat`、`ast`、`parser`       |
| 枚举值         | PascalCase   | `Red`、`Green`、`Blue`         |
| 类型参数       | 单个大写字母 | `T`、`U`、`V`                  |

---

## 11. 现有代码一致性

所有新代码必须与现有代码库的风格保持一致。当规范与现有代码冲突时，优先遵循现有代码的风格。
