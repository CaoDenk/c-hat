# C^ 可空类型设计文档

## 0. 设计哲学

C^ 的可空类型是**编译器内置的语言特性**，不依赖任何标准库。

> 语言核心特性不应对标准库开洞。任何人都可以实现自己的标准库，不能让编译器为 `Optional` 特殊开后门。

这与 C^ 异常处理的设计哲学一致：异常可以是任何类型，不绑定 `std.Exception`。

`?` 可空类型和标准库的 `Optional<T>` **是两个独立的东西**：
- `T?` 是内置语法，编译器原生支持，离开标准库照样能用
- `Optional<T>` 是标准库用 C^ 写的泛型类，是对 `T?` 的封装（可选使用）

---

## 1. 基本语法

### 1.1 可空类型声明

`?` 是后缀类型修饰符，将任意类型包装为"可能为空"的版本：

```cpp
int?       // 可空整数
Node?^     // 可空的 Node 指针（即：指针可能不存在）
string?    // 可空字符串
int?[]     // 可空切片
```

### 1.2 `null` 字面量

`null` 是内置关键字，只能赋给 `T?` 类型，不能赋给普通类型：

```cpp
int^ p = null;     // Error: 非空指针不接受 null
int?^ p = null;    // OK

int x = null;      // Error: int 不是可空类型
int? x = null;     // OK
```

### 1.3 赋值与构造

```cpp
int? a = 42;       // 有值：直接赋值，编译器将 42 包装为有值状态
int? b = null;     // 无值

Node?^ ptr = ^node;   // 有值：包装非空指针
Node?^ ptr = null;    // 无值
```

## 2. 使用方式

### 2.1 `if` 解包（推荐）

在 `if` 条件中，`T?` 会自动检查是否有值。若有值，在 `if` 块内变量**自动解包**为非空类型：

```cpp
int? x = get_value();

if (x) {
    // 此块内 x 的类型是 int（已解包，非空）
    print(x + 1);  // 直接使用，不需要额外操作
}
```

指针的解包：

```cpp
Node?^ node = find_node(42);

if (node) {
    // 此块内 node 的类型是 Node^（已解包）
    node->value = 100;
}
```

### 2.2 `match` 模式匹配

```cpp
Node?^ result = find_node(42);

match (result) {
    case ?node => node->value = 100,   // 有值时，node 绑定为 Node^
    case null  => print("not found"),
}
```

### 2.3 `?` 传播运算符（链式短路）

在返回 `T?` 或 `Result<T,E>` 的函数中，`?` 后缀在值为空/失败时**提前 return**：

| 表达式类型 | `?` 的行为 | 要求当前函数返回类型 |
|---|---|---|
| `T?` | 为 null 时 `return null` | 必须是某 `U?` |
| `Result<T,E>` | 为 Err(e) 时 `return Err(e)` | 必须是 `Result<U,E>`（E 兼容） |
| 其他类型 | **编译错误** | — |

```c^
func get_grandchild(Node?^ root) -> Node?^ {
    var child = root?->next;   // root 为 null 则函数立即 return null
    return child?->next;       // child->next 为 null 则 return null
}

func read_config() -> Result<Config, Error> {
    var text = read_file("cfg.txt")?;  // Err 时 return Err(e)
    var cfg  = parse_config(text)?;    // Err 时 return Err(e)
    return cfg;
}

// 在 void/int 返回函数里用 ? → 编译错误
func bad() -> int {
    var x = maybe_null()?;  // Error: ? 要求函数返回 T? 或 Result<T,E>
}
```

### 2.4 `??` 空合并运算符

```c^
var actual = get_count() ?? 0;           // 为 null 时使用默认值
var node   = find_node(key) ?? default_node;
```

### 2.5 强制解包（`!` 后缀）

当你确定值不为 null 时，用 `!` 强制解包。

**规则：**
- `expr!` 要求 `expr` 的类型必须是 `T?` 或 `Result<T,E>`，否则**编译错误**
- 运行时若值为 null / Err，触发 **panic**，并携带**源文件路径 + 行号**
- 编译器能静态证明不为 null 时，自动消除运行时检查（零开销）

```c^
var int? maybe = get_value();
var x = maybe!;         // 成功：x 类型为 int；失败：panic at main.ch:42

var Result<int, Error> r = parse("123");
var v = r!;             // 成功：v 类型为 int；失败：panic at main.ch:45

// 非可空类型使用 ! → 编译错误（类型前置，编译器已知类型）
var int n = 5;
n!;                     // Error: int 不是可空类型，不能使用 !
```

**panic 信息格式：**
```
panic: null assertion failed at src/main.ch:42
panic: unwrap failed (Err) at src/parser.ch:18
```

**`?` 与 `!` 的配合：**
```c^
func load_user(int id) -> User? {
    var row  = db.query(id)?;   // 找不到则传播 null（预期情况）
    var user = parse(row)!;     // 解析失败是 bug，panic（非预期）
    return user;
}
```

> 优先用 `if`/`match` 安全解包，`!` 是"我确定不会 null"的断言，用错了 panic 是你的责任。

## 3. 类型组合

`?` 后缀修饰它左边的**整个类型表达式**（区别于 `!` 的左绑定规则）：

```
类型写法     语义
──────────────────────────────────────────────────
int?         可空整数值
int?^        可空指针（Optional 包裹 int^）
int^?        同上，? 修饰 int^（等价写法，推荐 int?^）
int?[]       可空切片
int!?        指向不可变 int 的可空值（罕见）
Node?^!      不可变的可空指针（指针本身不可重新绑定）
```

> **推荐写法**：`?` 紧跟被修饰的核心类型，如 `Node?^` 而非 `Node^?`，可读性更好。

## 4. 编译器行为

### 4.1 内存表示

`T?` 的内存布局由编译器决定，针对不同类型做优化：

- **指针类型 `T?^`**：直接用空指针（null = 0）表示空状态，**零额外开销**（与 C 的 nullable 指针完全等价）
- **值类型 `int?`、`float?` 等**：编译器生成一个额外的 `bool` 标志位（或利用 NaN/哨兵值优化）
- **枚举/有限类型**：编译器可能利用未使用的枚举值存储空状态

这是编译器的内部优化，对语言用户透明。

### 4.2 `if` 解包的实现

```cpp
// 源代码
if (node) {
    node->value = 100;
}

// 编译器等价展开（伪代码）
if (node != null) {
    Node^ __unwrapped_node = node;  // 内部创建解包后的非空绑定
    __unwrapped_node->value = 100;
}
```

### 4.3 与非空类型的关系

非空类型 `T^` 是 `T?^` 的子类型，可以隐式转换：

```cpp
Node^ non_null = new Node();
Node?^ nullable = non_null;   // OK：非空可以隐式转换为可空
// Node^ back = nullable;     // Error：可空不能隐式转换为非空（需要解包）
```

## 5. 与标准库 `Optional<T>` 的关系

标准库可以提供 `Optional<T>`，它是用 C^ 写的普通泛型类，**不需要编译器开任何特殊后门**：

```cpp
// 标准库的实现示意（完全用 C^ 写，不依赖编译器特权）
class Optional<T> {
    private T? _value;    // 内部用内置 T? 存储

    public Optional(T value) { _value = value; }
    public Optional()        { _value = null; }

    public get HasValue => _value != null;
    public func value() -> T { return _value!; }
    public func value_or(T default_val) -> T { return _value ?? default_val; }
}
```

**关系总结**：
- `T?` 是语言内置，编译器直接支持，无需标准库
- `Optional<T>` 是标准库提供的封装，底层用 `T?` 实现
- 用户可以完全绕过 `Optional<T>`，直接用 `T?`
- 用户也可以自己实现 `Optional<T>` 或类似的工具类，行为完全等价

## 6. 与 `Result<T, E>` 的分工

| | `T?` | `Result<T, E>` |
|--|--|--|
| 语义 | 值可能不存在（无原因） | 操作可能失败（有原因） |
| 空/失败信息 | 无 | 携带错误类型 `E` |
| 适用场景 | 查找、可选字段、可空参数 | IO、解析、网络、业务逻辑 |
| 传播运算符 | `?`（提前返回 null） | `?`（提前返回 Err） |
| 是否内置 | ✅ 编译器内置 | ✅ 编译器内置 |

```cpp
// 查找：用 T?（"找不到"本身就是完整的信息）
func find(int key) -> Node?^;

// 文件读取：用 Result（需要知道为什么失败）
func read_file(string path) -> Result<string, IOError>;
```

## 7. 与 C++ 的对照

| C^ | C++ 等价 | 说明 |
|--|--|--|
| `int?^` | `int*`（靠约定）或 `gsl::owner<int*>` | C++ 无内置空安全保证 |
| `int^`（非空） | 无原生等价（靠 `gsl::not_null`） | C^ 类型系统层面保证 |
| `if (p) { p->x; }` | `if (p) { p->x; }` | 相同模式 |
| `p ?? val` | `p ? *p : val` | C^ 运算符更简洁 |
| `p!` | `*p`（不检查）| C^ 明确标记"我知道风险" |
