# 链接器实现原理文档

## 概述
本文档详细描述了 C hat 编译器如何使用 LLD 库实现链接功能，将 LLVM 生成的目标文件链接为可执行程序。

## 背景与目标
在 C hat 编译器开发过程中，我们尝试了多种链接方案：
1. **使用 LLD 库直接链接** - 初次尝试，遇到参数配置问题
2. **使用 clang 命令进行链接** - 依赖外部工具
3. **使用 lld-link 可执行文件进行链接** - 调试参数配置
4. **最终方案：使用 LLD 库直接链接** - 完全集成到编译器

## 问题分析

### 主要问题
在最初尝试使用 LLD 库链接时遇到的问题：
- 无法找到 printf 等 C 标准库函数符号
- 库文件加载冲突（同时使用 libcmt 和 msvcrt）
- 链接器参数配置需要与 lld-link 可执行文件完全一致

### 关键发现
通过使用 lld-link 可执行文件调试，我们找到了正确的参数配置，然后将这些参数应用到 LLD 库中，最终成功实现链接。

## 最终解决方案

### 1. 使用 LLD 库直接链接

我们选择使用 LLD 库，而不是外部可执行文件，原因如下：
- 完全集成到编译器中，不需要外部依赖
- 与编译器共享相同的 LLVM 基础设施
- 更好的性能和控制
- 符合用户需求："不要调用外部工具，将链接集成到编译器上"

### 2. 链接器参数配置

#### 核心参数配置（在 main.cpp 中实现）：
```
lld-link
/OUT:test_printf.exe
/DEFAULTLIB:libcmt
/DEFAULTLIB:oldnames
/LIBPATH:<MSVC 库路径>
/LIBPATH:<UCRT 库路径>
/LIBPATH:<Windows SDK 库路径>
test_printf.obj
```

#### 关键参数解释：

- `lld-link`（第一个参数）
  - 程序名，LLD 要求第一个参数必须是程序名

- `/DEFAULTLIB:libcmt`
  - 使用 libcmt.lib 作为 C 标准库（静态链接版本）

- `/DEFAULTLIB:oldnames`
  - 处理旧的 C 库函数名称映射

- `/LIBPATH`
  - 指定库文件搜索路径

### 3. 避免库冲突

最初发现 `--c-lib-file` 的默认值是 "msvcrt.lib"，这导致了库冲突，因为同时使用了 `/DEFAULTLIB:libcmt` 和显式添加 msvcrt.lib。

**解决方案**：
```cpp
argParser.add_argument("--c-lib-file")
    .help("C standard library file to link")
    .default_value(std::string("")); // 空字符串作为默认值
```

### 4. 完整的链接流程

1. **生成目标文件**：使用 LLVM 代码生成器将 C hat 源代码编译为目标文件
2. **构建链接参数**：在 main.cpp 中构建 LLD 链接参数
3. **执行链接**：通过 `lld::lldMain()` 调用 LLD 库
4. **生成可执行文件**：链接成功后，生成可执行文件

## 实现细节

### main.cpp 的主要修改

在 main.cpp 中实现的关键代码片段：

1. **LLD 头文件和驱动声明**：

```cpp
// LLD headers
#include <lld/Common/Driver.h>
#include <llvm/ADT/ArrayRef.h>
#include <llvm/Support/raw_ostream.h>

// Declare that we use the COFF driver
LLD_HAS_DRIVER(coff);
LLD_HAS_DRIVER(elf);
LLD_HAS_DRIVER(mingw);
LLD_HAS_DRIVER(macho);
LLD_HAS_DRIVER(wasm);
```

2. **链接器参数准备**：

```cpp
std::vector<std::string> argsStr;
argsStr.push_back("lld-link");
argsStr.push_back(outArg);
argsStr.push_back("/DEFAULTLIB:libcmt");
argsStr.push_back("/DEFAULTLIB:oldnames");

// 添加库文件路径 - 使用与 clang 相同的路径
std::vector<std::string> libPaths = {
    "C:/Program Files/Microsoft Visual Studio/2022/Community/VC/Tools/MSVC/14.44.35207/lib/x64",
    "C:/Program Files (x86)/Windows Kits/10/Lib/10.0.26100.0/ucrt/x64",
    "C:/Program Files (x86)/Windows Kits/10/Lib/10.0.26100.0/um/x64"};

for (const auto &path : libPaths) {
  argsStr.push_back("/LIBPATH:" + path);
}

argsStr.push_back(objOutputFile);
```

3. **链接器准备和执行部分**：

```cpp
// 3. 使用 LLD 库进行链接
std::println("\nLinking with LLD (library)...");

// 构建参数数组
std::vector<const char *> args;
for (const auto &s : argsStr) {
  args.push_back(s.c_str());
}

// 定义驱动程序
std::vector<lld::DriverDef> drivers = LLD_ALL_DRIVERS;

// 调用 LLD main
lld::Result result =
    lld::lldMain(llvm::ArrayRef<const char *>(args.data(), args.size()),
                 llvm::outs(), llvm::errs(), drivers);

if (result.retCode == 0) {
  std::println("\n✓ Executable generated: {}", exeOutputFile);
} else {
  std::println("\n✗ Linking failed with code: {}", result.retCode);
}
```

### 可执行程序名称

编译器的可执行程序名称已从 `c_hat_parser` 改为 `c_hat_compiler`，更符合项目定位。

使用 LLD 库直接链接的方案是成功的：

- **支持调用 printf 等变参函数**
